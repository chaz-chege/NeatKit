What is Base64?
Base64 encodes binary data as ASCII text using 64 characters (A–Z, a–z, 0–9, +, /). Used in emails (MIME), HTML image embedding (data:image/png;base64,…), JSON payloads, and API authentication.
URL-safe Base64
+ and / have special meaning in URLs. URL-safe mode replaces them with - and _ and removes = padding. Use for JWT tokens, URL parameters, and filenames.
Encoding ≠ Encryption
Base64 is not encryption. Anyone can decode it instantly. It changes the format of data, not its security. Never use it to hide passwords or sensitive data.
MIME 76-char lines
RFC 2045 requires Base64 in emails to be split into 76-character lines separated by CRLF. Enable "76-char lines" when preparing Base64 for email attachments or MIME content.
