About this tool

How do I merge PDF files without uploading them?
Neatkit's PDF merger uses pdf-lib, a JavaScript library that processes PDFs entirely in your browser using WebAssembly. Your files are loaded into your device's memory, merged, and downloaded — never sent to a server.
What is the maximum PDF file size?
The practical limit depends on your device's available RAM. Most devices handle PDFs up to 200-300MB per file comfortably. Very large files (500MB+) may be slow on older devices.
Will the merged PDF look identical to the originals?
Yes. pdf-lib copies pages directly without re-rendering, so all fonts, images, vector graphics, and formatting are preserved exactly.
Can I reorder pages when merging?
Yes — drag the files in the list to reorder them before merging. The output PDF will follow that exact order.

Tips for merging PDFs

Drag to reorder before merging. The final PDF pages follow the file order in the list. Drag files into the correct sequence — you can't reorder pages after merging without splitting and re-merging.
Check file sizes first. Merging 10 large PDFs can produce a very large output file. If your merged PDF needs to stay under a portal's upload limit, compress the individual PDFs or the source images before merging.
Password-protected PDFs won't merge. If a PDF is password protected, you'll need to unlock it first. This tool works with standard, unprotected PDF files.

How PDF merging works

This tool merges multiple PDF files into one entirely in your browser using PDF-lib, an open-source JavaScript library. Your files are never uploaded to any server — they're read locally using the FileReader API and combined in memory, then downloaded as a single PDF file.

The merge process reads each PDF's page structure, copies all pages into a new document in the order you specify, and saves the result. The output PDF preserves the original page dimensions and content of each source file — nothing is re-encoded or compressed.
