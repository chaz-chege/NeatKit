When to use each case format

camelCase and PascalCase are programming conventions. Use camelCase for variable names in JavaScript, Java, and Swift. Use PascalCase for class names and component names in most languages.
snake_case is the standard in Python for variables and function names. CONSTANT_CASE (also called SCREAMING_SNAKE_CASE) is used for constants and environment variables.
kebab-case is used in HTML attributes, CSS class names, and URL slugs. It's the most readable in contexts where underscores aren't allowed or look odd.

A guide to text case formats

Different contexts have different conventions for how text should be formatted. Understanding when each case is appropriate saves time and avoids style inconsistencies across your work.

⌨️
UPPERCASE
All letters capitalised. Used for acronyms, headings that demand attention, button labels, and abbreviated terms. Avoid for long passages — it reads as shouting.
✍️
Title Case
First letter of each major word capitalised. Standard for book titles, article headlines, product names, and formal document headings.
💻
camelCase
First word lowercase, subsequent words capitalised, no spaces. Standard for variable names, function names, and JSON keys in JavaScript, Java, and Swift.
🐍
snake_case
All lowercase with underscores between words. The Python standard for variables, functions, and module names. Also common in database column names and file names.
🌐
kebab-case
All lowercase with hyphens between words. Used in HTML class and ID attributes, CSS property names, URL slugs, and command-line tool options.
🏛️
PascalCase
Every word capitalised, no spaces. Standard for class names in most object-oriented languages, React component names, and TypeScript interfaces.

Case conventions by language and context

🐍
Python
Variables and functions: snake_case. Classes: PascalCase. Constants: CONSTANT_CASE. Modules and packages: snake_case. Private methods: _snake_case.
🟨
JavaScript / TypeScript
Variables and functions: camelCase. Classes and components: PascalCase. Constants: CONSTANT_CASE. CSS classes: kebab-case. File names: kebab-case or camelCase.
☕
Java / Kotlin
Variables and methods: camelCase. Classes: PascalCase. Constants: CONSTANT_CASE. Packages: all lowercase with dots. Android resources: snake_case.
🌐
HTML / CSS
CSS class names: kebab-case. HTML IDs: kebab-case. CSS custom properties: --kebab-case. BEM methodology uses block__element--modifier in kebab-case.
🗄️
Databases / SQL
Table names: snake_case or PascalCase depending on convention. Column names: snake_case (PostgreSQL, MySQL standard). SQL keywords: UPPERCASE by convention.
📁
File names and URLs
URLs and slugs: kebab-case (SEO-friendly and readable). File names: kebab-case or snake_case. Avoid spaces and UPPERCASE in file names for cross-platform compatibility.

Frequently asked questions

Both formats join words without spaces and capitalise each word — the only difference is the first word. camelCase starts with a lowercase letter (helloWorld). PascalCase starts with an uppercase letter (HelloWorld). camelCase is used for variable and function names; PascalCase is used for class and component names in most programming conventions.
Programming languages use spaces as delimiters to separate tokens — keywords, operators, and identifiers. A variable name with a space would be interpreted as two separate tokens, causing a syntax error. That's why developers use camelCase, snake_case, or kebab-case to create multi-word identifiers without spaces.
kebab-case is the standard for URLs and is preferred by Google. For example: yoursite.com/blog/how-to-compress-images is better than yoursite.com/blog/howToCompressImages or yoursite.com/blog/how_to_compress_images. Hyphens are treated as word separators by search engines; underscores are not.
Numbers are preserved as-is in the output. Special characters like hyphens, underscores, and dots are treated as word separators when converting between formats. Other special characters (punctuation, symbols) are carried through to the output without modification.
CONSTANT_CASE (also called SCREAMING_SNAKE_CASE) is used for constants — values that don't change during program execution. Examples include environment variable names (DATABASE_URL, API_KEY), configuration constants, and named magic numbers. The uppercase letters signal to other developers that this value should not be reassigned.
