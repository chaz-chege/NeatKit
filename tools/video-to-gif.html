Tips for smaller, better-looking GIFs

Keep clips under 10 seconds. GIF file size grows linearly with duration. A 3-second clip at 12fps is ~36 frames. A 30-second clip is 360 frames — 10× the data. Trim aggressively and loop the interesting part rather than making a long clip.
480px wide is the sweet spot. Most platforms display GIFs at 400–600px wide. Using the original resolution for a 1080p video produces a massive GIF with no visible improvement at display size. 480px looks identical in a chat or on a website but is a fraction of the size.
12fps is intentional, not a limitation. The GIF format looks and feels natural at 10–15fps — it has its own aesthetic that's distinct from smooth video. Going above 20fps adds significant file size with minimal visual improvement in most cases.

How two-pass palette optimisation works

Most basic video-to-GIF tools do a single pass: convert each frame directly to the 256-colour GIF palette using a generic approximation. This produces washed-out colours, especially in videos with gradients or subtle colour variation.

This tool uses FFmpeg's two-pass palette approach — the same method used by professional GIF creators:

🎨
Pass 1 — palettegen
FFmpeg analyses every frame in the clip and calculates the optimal 256-colour palette specifically for this video's content. A video with a lot of blues and greens gets a palette biased toward those colours.
✨
Pass 2 — paletteuse
FFmpeg re-encodes the video using the optimised palette. Each pixel is mapped to the closest colour in the custom palette rather than a generic one, producing noticeably richer, more accurate colours.
📉
Dithering
Since GIF supports only 256 colours, gradients have to be approximated. Dithering adds controlled noise patterns that fool the eye into perceiving smooth colour transitions. Bayer dithering is a good default.
🔢
Frame reduction
The fps setting controls how many frames per second are sampled from the source video. Fewer frames = smaller file. 12fps is a good default — the motion looks smooth enough and file size stays manageable.

GIF size limits by platform

If you're making a GIF to share somewhere specific, here are the limits you need to stay under:

💬
Discord
8MB for free users. Nitro users get 50MB. GIFs in messages must be under 8MB to send without a link.
💼
Slack
No hard GIF limit for inline display, but files over 1GB can't be uploaded. Animated GIFs in messages display automatically up to ~50MB.
🐦
X / Twitter
15MB on web, 5MB on mobile. Twitter converts GIFs to MP4 for playback anyway, so smaller is always better for upload speed.
📸
Giphy / Tenor
100MB upload limit on Giphy. Tenor accepts up to 100MB. Both re-process GIFs after upload so the original quality is what matters.

Frequently asked questions

The first pass analyses your video to build an optimal colour palette for your specific clip. The second pass uses that palette to encode each frame. You could skip the first pass and use a generic palette, but the colour quality is noticeably worse — especially for videos with subtle gradients or complex colour. Two passes take longer but produce a much better result.
GIF size is mainly determined by: clip duration × frame rate × resolution. Reduce any of these to shrink the file. The biggest win is usually reducing width — try 320px or 480px instead of the original resolution. Next, reduce the clip duration. Then reduce fps to 8–10. The colour depth setting (64/128/256) has a smaller effect than those three.
Bayer dithering (the default) works well for most content — it adds a regular grid pattern that's subtle and effective. Floyd-Steinberg produces smoother gradients but can look grainy on some content and produces larger files. None (no dithering) gives the smallest file but looks blocky on anything with gradients. Try Bayer first; switch to Floyd-Steinberg if gradients look bad.
